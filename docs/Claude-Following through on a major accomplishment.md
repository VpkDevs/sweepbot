### KNOWN EDGE CASES & GOTCHAS

1. **Platform API Changes:** Sweepstakes platforms change their APIs frequently. If session capture breaks, priority 1 is fixing the adapter.

2. **Credential Security:** Users will ask "is my password safe?" Answer: Yes, it's AES-256 encrypted and never leaves the device unencrypted. Make this very clear in UI.

3. **Browser Storage Limits:** Extension local storage has limits (~10MB). If user data grows large, implement cleanup (keep last 90 days, archive older).

4. **Cross-Domain XHR Interception:** Some platforms may block cross-origin requests. If session capture stops working, may need Service Worker approach instead.

5. **RTP Calculation Variance:** User RTP won't match platform RTP perfectly — sample size matters. Be clear about confidence intervals.

6. **Redemption Delays:** Some platforms take weeks to process redemptions. Manage expectations; don't over-promise on speed.

7. **Responsible Play Enforcement:** If user sets a loss limit and hits it, actually enforce it — don't let them dismiss it.
